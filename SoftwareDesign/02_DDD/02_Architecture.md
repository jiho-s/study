# Chapter 2 아키텍처 개요

## 2.1 네 개의 영역

표현, 응용, 도메인, 인프라스트럭처

표현 - Controller

응용 - Service

도메인 - Entity, Value

인프라스트럭처 - 카프카 등

## 2.2 계층 구조 아키텍처

표현 -> 응용 -> 도메인 -> 인프라스트럭처

상위에서 하위 계층으로만 의존한다

## 2.4 도메인 영역의 주요 구성요소

### 2.4.1 엔티티와 벨류

엔티티는 단순히 데이터를 담고 있는 데이터 구조라기보다는 데이터와 함께 기능을 제공하는 객체이다.

### 2.4.2 애그리거트

도메인 모댈에서 전체 구조를 이해하는데 도움이 되는 것 에그리거트는 관련 객체를 하나로 묶은 군집

### 2.4.3 리포지토리

리포지토리는 애그리거트 단위로 도메인 객체를 저장하고 조회하는 기능을 정의한다
